rubydo

Given:

∴ cat test.rb puts "\t#{RUBY_PATCHLEVEL} #{RUBY_RELEASE_DATE} #{RUBY_VERSION} #{RUBY_PLATFORM}" ∴ rvm list ruby: - ruby-1.8.6-24700 - ruby-1.8.6-head - ruby-1.8.6-p111 - ruby-1.8.6-p369 - ruby-1.8.7-p174 - ruby-1.9.1-p243 - ruby-1.9.2-preview1 jruby: - jruby-1.3.1 ree: - ruby-enterprise-1.8.6-20090610 system: - (ruby 1.8.7 (2008-08-11 patchlevel 72) [universal-darwin10.0])

We can run the ruby program 'test.rb' against all versions of ruby installed by rvm very simply:

∴ rvm rubydo -f test.rb ruby-1.8.6-24700: 387 2009-08-26 1.8.6 i686-darwin10.0.0 ruby-1.8.6-head: 387 2009-08-26 1.8.6 i686-darwin10.0.0 ruby-1.8.6-p369: 369 2009-06-08 1.8.6 i686-darwin9.8.0 ruby-1.8.7-p174: 174 2009-06-12 1.8.7 i686-darwin9.8.0 ruby-1.9.1-p243: 243 2009-07-16 1.9.1 i386-darwin9.8.0 ruby-1.9.2-preview1: -1 2009-07-18 1.9.2 i386-darwin10.0.0 ruby-enterprise-1.8.6-20090610: 287 2008-08-11 1.8.6 i686-darwin10.0.0 ruby-jruby-1.3.1: 287 2009-06-15 1.8.6 java

If we want we can also restrict to a subset of named top level selections:

∴ rvm rubydo 1.9,1.8,1.8.7,jruby,ree -f test.rb ruby-1.9.1-p243: 243 2009-07-16 1.9.1 i386-darwin9.8.0 ruby-1.8.6-p369: 369 2009-06-08 1.8.6 i686-darwin9.8.0 ruby-1.8.7-p174: 174 2009-06-12 1.8.7 i686-darwin9.8.0 ruby-jruby-1.3.1: 287 2009-06-15 1.8.6 java ruby-enterprise-1.8.6-20090610: 287 2008-08-11 1.8.6 i686-darwin10.0.0
